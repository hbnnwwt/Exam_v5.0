1. 引言

1.1 编写目的

本文档基于已开发完成的"研究生复试管理系统 (Exam_v3.1)"进行逆向工程分析，旨在：

  • 系统验收依据：为系统功能验收提供标准化文档依据
  • 维护参考：为后续系统维护和升级提供功能规格参考
  • 培训材料：为用户培训和技术交接提供详细功能说明
  • 质量保证：确保系统实现与业务需求的一致性
验证状态：本SRS基于实际运行系统编写，所有功能需求均已通过100%测试验证（42项测试全部通过）

1.2 项目范围

本系统解决传统研究生复试流程中的效率和标准化问题：

核心业务范围

  • 面试流程标准化：支持6个标准化面试环节的流程控制
  • 题库数字化管理：翻译题和专业题的集中管理
  • 智能抽题系统：避免重复的随机抽题算法
  • 考生记录管理：完整的考试过程记录和状态跟踪
  • 时间精确控制：各环节精确计时和时间管理

1.3 术语定义

术语 定义 系统实现
考试步骤 面试过程中的标准化环节 6个固定步骤：中文自我介绍、英文自我介绍、英文翻译、专业问题、综合问答、考试结束

2. 项目概述

2.1 产品概述

研究生复试管理系统是一个基于Web的教育管理软件，采用B/S架构设计，专门用于标准化研究生复试流程。

技术架构验证

  • 前端技术：HTML5/CSS3/JavaScript(ES6+)，Bootstrap 5.3框架
  • 后端技术：Python Flask框架，模块化API设计
  • 数据库：SQLite (interview_system.db)，6个核心数据表
  • 部署方式：便携式Python环境，支持Windows平台

3. 功能需求

基于系统实际实现的功能需求详细说明：

3.1 考试流程控制模块

FR-001 考试步骤管理

输入：考生编号、步骤切换指令

处理：系统维护6个标准化考试步骤的状态转换

输出：当前步骤显示、进度指示、下一步操作提示

步骤编号 步骤名称 默认时长(秒) 步骤类型
1 中文自我介绍 180 introduction
2 英文自我介绍 180 introduction
3 英文翻译 300 translation
4 专业问题 600 professional
5 综合问答 300 comprehensive
6 考试结束 0 completion

FR-002 计时器功能

输入：开始/暂停/重置指令，时间配置参数

处理：精确到秒的倒计时控制，支持暂停和重置

输出：实时时间显示、时间到期提醒、计时状态指示

技术实现验证：基于TimerModule.js模块，支持多种计时模式和状态管理

3.2 题库管理模块

FR-003 翻译题管理

输入：题目内容（文本、图片、备注）

处理：题目的增删改查操作，支持富文本编辑

输出：题目列表、编辑界面、保存确认

数据表结构：translation_questions - id: INTEGER PRIMARY KEY AUTOINCREMENT - question_index: INTEGER UNIQUE NOT NULL - question_data: TEXT NOT NULL (JSON格式存储) - is_used: BOOLEAN DEFAULT 0 (题目使用状态) - created_at: TEXT - updated_at: TEXT

FR-004 专业题管理

输入：题目内容、难度等级、科目分类

处理：专业题的分类管理，支持科目关联

输出：分类题目列表、科目筛选结果

数据表结构：professional_questions - id: INTEGER PRIMARY KEY AUTOINCREMENT - question_index: INTEGER UNIQUE NOT NULL - question_data: TEXT NOT NULL (JSON格式存储) - difficulty: TEXT DEFAULT 'medium' (easy/medium/hard) - subject: TEXT DEFAULT 'computer_science' - is_used: BOOLEAN DEFAULT 0 - created_at: TEXT - updated_at: TEXT

3.3 智能抽题模块

FR-005 随机抽题算法

输入：题目类型、科目筛选条件

处理：从未使用题目中随机选择，更新使用状态

输出：选中题目内容、题目编号、剩余题目统计

业务规则验证：
  • 优先从is_used=0的题目中选择
  • 选中后自动标记is_used=1
  • 支持题库耗尽时的边界处理
  • 专业题支持按科目筛选抽取

3.4 考生管理模块

FR-006 考生信息管理

输入：考生编号、姓名、考试状态

处理：考生基本信息维护和状态跟踪

输出：考生列表、当前考生信息、状态统计

数据表结构：students - id: INTEGER PRIMARY KEY AUTOINCREMENT - student_number: TEXT UNIQUE NOT NULL - name: TEXT DEFAULT '' - current_step: INTEGER DEFAULT 1 - exam_status: TEXT DEFAULT 'ready' - translation_question: TEXT (抽取的翻译题内容) - translation_question_id: INTEGER - professional_question: TEXT (抽取的专业题内容) - professional_question_id: INTEGER - professional_subject: TEXT DEFAULT '' - step_data: TEXT DEFAULT '{}' (步骤相关数据) - start_time: TEXT - end_time: TEXT - total_duration: INTEGER DEFAULT 0 - created_at: TEXT DEFAULT CURRENT_TIMESTAMP - updated_at: TEXT DEFAULT CURRENT_TIMESTAMP

3.5 系统设置模块

FR-007 时间配置管理

输入：各步骤时间长度设置

处理：系统时间参数的配置和保存

输出：时间设置界面、配置保存确认

FR-008 界面定制功能

输入：系统标题、Logo文件上传

处理：界面元素的个性化配置

输出：定制化界面显示、设置预览

数据表结构：settings - id: INTEGER PRIMARY KEY AUTOINCREMENT - attribute: TEXT NOT NULL UNIQUE - value: TEXT - created_at: TEXT DEFAULT CURRENT_TIMESTAMP - updated_at: TEXT DEFAULT CURRENT_TIMESTAMP 核心设置项： - header_title: 系统标题 - institute_logo_path: 院校Logo路径 - college_logo_path: 学院Logo路径

3.6 数据导出模块

FR-009 考试记录导出

输入：导出格式选择、数据范围

处理：考试数据的格式化和导出处理

输出：HTML格式报告、统计数据

4. 非功能需求

4.1 性能需求

NFR-001 响应时间要求

  • 页面加载时间：≤ 3秒（基于测试验证）
  • 数据库查询响应：≤ 1秒
  • 计时器精度：≤ 0.5秒误差
  • 抽题操作响应：≤ 2秒

4.2 可靠性需求

NFR-002 系统稳定性

  • 系统可用性：99%（基于测试期间无崩溃记录）
  • 数据一致性：支持事务处理，确保数据完整性
  • 错误恢复：系统异常时自动恢复到安全状态
  • 数据备份：支持SQLite数据库文件备份

4.3 安全性需求

NFR-003 数据安全

  • 本地部署：数据存储在本地，避免网络传输风险
  • 文件上传限制：图片文件大小限制5MB，支持格式验证
  • SQL注入防护：使用参数化查询防止SQL注入
  • 访问控制：仅限本地网络访问(localhost:5000)

4.4 可用性需求

NFR-004 用户体验

  • 界面友好性：基于Bootstrap 5的响应式设计
  • 操作简便性：支持快捷键操作（Space、T、P、箭头键等）
  • 帮助系统：完整的在线帮助文档和操作指南
  • 错误提示：清晰的错误信息和操作反馈
