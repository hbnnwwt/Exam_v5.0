# 抽题动画优化设计文档

**项目**: 研究生复试系统
**日期**: 2026-03-19
**类型**: 功能优化

---

## 1. 需求概述

优化考试抽题动画效果，参考旧版实现并增加新效果，提升用户体验。

## 2. 设计方案

### 2.1 动画逻辑

采用旧版转圈逻辑（兼容现有API时序）：

| 参数 | 值 |
|------|-----|
| 转圈次数 | 3圈 |
| 间隔时间 | 100ms/80ms/60ms（递减） |
| 动画总时长 | ~2.4秒（10个题目） |

**动画流程**：
1. 点击"开始抽取"后，立即发起API请求获取随机题目
2. 同时开始扫描动画（与现有逻辑相同）
3. API返回后，计算还需转几圈然后停在目标位置（兼容现有逻辑）
4. 每圈逐渐加快速度（100ms → 80ms → 60ms）
5. 停在目标题目，触发弹跳效果

### 2.2 视觉设计

**扫描中样式**：
- 背景色：`#fed7aa`（浅橙色，与现有代码一致）
- 边框色：`#ffc107`（黄色）
- 缩放：`scale(1.1)`
- 脉冲动画：`pulse 0.15s`

**选中后样式**：
- 背景色：`#fd7e14`（橙色）
- 缩放：`scale(1.1)`
- 阴影：`0 4px 12px rgba(253, 126, 20, 0.6)`
- 弹跳效果：`bounce 0.5s`

### 2.3 脉冲动画

```css
@keyframes pulse {
  0% { transform: scale(1); }
  50% { transform: scale(1.15); }
  100% { transform: scale(1); }
}
```

**注意**：使用 `scale(1)` 基础值，避免与 `.selected` 的 `scale(1.1)` 冲突

### 2.4 弹跳动画

```css
@keyframes bounce {
  0% { transform: scale(1.1); }
  25% { transform: scale(1.2); }
  50% { transform: scale(1.05); }
  75% { transform: scale(1.15); }
  100% { transform: scale(1.1); }
}
```

**注意**：动画结束后保持 `scale(1.1)` 与 `.selected` 样式一致

## 3. 实施计划

### 3.1 修改文件

- `frontend/src/components/StepContent.vue`

### 3.2 具体改动

1. **动画逻辑重构** (`startDraw` 函数)
   - 改为固定转圈模式（3圈）
   - 速度递增：每圈减少20ms间隔（100ms → 80ms → 60ms）
   - 兼容现有API时序：先发请求，API返回后计算圈数停在目标位置

2. **新增CSS样式**
   - `.question-item.animating` - 扫描中高亮样式（脉冲动画）
   - `@keyframes pulse` - 脉冲动画
   - `@keyframes bounce` - 弹跳动画
   - `.question-item.bounce` - 弹跳效果类

3. **状态管理**
   - 保留现有 `tempHighlightId` 变量名
   - 新增 `isAnimating` 状态控制动画
   - 动画结束后触发弹跳效果，然后变为选中状态

## 4. 验收标准

- [x] 点击"开始抽取"后，题目列表开始转圈扫描
- [x] 扫描速度递增（3圈越来越快）
- [x] 第3圈结束后停在被抽取的题目
- [x] 扫描中有脉冲动画效果
- [x] 停止后有弹跳效果并保持橙色高亮
- [x] 动画总时长控制在1-2秒内
